# IAM Dataset Schema Definition

This document outlines the schema for the synthetic IAM (Identity and Access Management) dataset generated by the `generate_iam_data.py` script. The data is stored in Parquet format.

## 1. Identities (`identities.parquet`)
Represents the users (employees, contractors, service accounts) within the organization.

| Column Name | Data Type | Description |
|---|---|---|
| `identity_id` | String (UUID) | Unique identifier for the identity. |
| `identity_type` | String | Type of identity (e.g., 'Human', 'Service Account'). |
| `status` | String | Employment status (e.g., 'Active', 'Inactive', 'Leave'). |
| `location` | String | Physical location or office of the identity. |
| `cost_center` | String | Cost center code associated with the identity. |
| `department` | String | Department the identity belongs to (e.g., 'Engineering', 'HR'). |
| `job_title` | String | Job title of the identity. |
| `manager_id` | String (UUID) | `identity_id` of the identity's manager. Null for top-level execs. |

## 2. Applications (`applications.parquet`)
Represents the business applications managed by the IAM system.

| Column Name | Data Type | Description |
|---|---|---|
| `app_id` | String (UUID) | Unique identifier for the application. |
| `app_name` | String | Name of the application. |
| `business_criticality` | String | Criticality level (e.g., 'Low', 'Medium', 'High', 'Critical'). |
| `app_owner_id` | String (UUID) | `identity_id` of the application owner. |

## 3. Resources (`resources.parquet`)
Represents specific instances or environments within an application (e.g., Production DB, Dev Environment, Active Directory).

| Column Name | Data Type | Description |
|---|---|---|
| `resource_id` | String (UUID) | Unique identifier for the resource. |
| `app_id` | String (UUID) | Foreign key linking to the parent Application. |
| `iga_source_name` | String | Name of the source system in the IGA tool. |
| `connection_type` | String | Method of integration (e.g., 'Direct', 'JDBC', 'API'). |
| `resource_type` | String | Type of resource (e.g., 'Platform', 'Application'). Platforms (like AD) typically have more complex entitlement structures. |

## 4. Accounts (`accounts.parquet`)
Represents the actual accounts (login credentials) existing on target resources.

| Column Name | Data Type | Description |
|---|---|---|
| `account_id` | String (UUID) | Unique identifier for the account on the target system. |
| `identity_id` | String (UUID) | Foreign key linking the account to an Identity (Owner). |
| `resource_id` | String (UUID) | Foreign key linking the account to a Resource. |
| `account_name` | String | Login name or username (e.g., 'acc_12345678'). |
| `is_privileged` | Boolean | Flag indicating if the account has privileged access. |
| `status` | String | Status of the account (e.g., 'Active'). |

## 5. Entitlements (`entitlements.parquet`)
Represents the permissions, roles, or groups available within a resource.

| Column Name | Data Type | Description |
|---|---|---|
| `entitlement_id` | String (UUID) | Unique identifier for the entitlement. |
| `resource_id` | String (UUID) | Foreign key linking the entitlement to a Resource. |
| `entitlement_name` | String | Name of the entitlement (e.g., 'Ent_ADMIN_1234'). |
| `is_requestable` | Boolean | Flag indicating if users can request this entitlement via self-service. |
| `owner_id` | String (UUID) | `identity_id` of the entitlement owner. |

## 6. Account Entitlements (`account_entitlements.parquet`)
Represents the many-to-many relationship between Accounts and Entitlements (i.e., "Who has what access").

| Column Name | Data Type | Description |
|---|---|---|
| `identity_id` | String (UUID) | Foreign key to Identity (denormalized for convenience). |
| `account_id` | String (UUID) | Foreign key to the Account holding the entitlement. |
| `entitlement_id` | String (UUID) | Foreign key to the Entitlement being held. |
| `assignment_type` | String | How the access was assigned (e.g., 'Birthright', 'Adhoc_Anomaly'). |
| `grant_date` | Date | Date when the entitlement was granted. |

## 7. Entitlement Groups (`entitlement_groups.parquet`)
Represents logical groupings of identities, often used for role-based access control (RBAC) or organizational hierarchy.

| Column Name | Data Type | Description |
|---|---|---|
| `ent_group_id` | String (UUID) | Unique identifier for the group. |
| `group_name` | String | Name of the group (e.g., 'Dept_Engineering', 'Team_Backend'). |
| `description` | String | Description of the group's purpose. |
| `owner_id` | String (UUID) | `identity_id` of the group owner. |
| `type` | String | Category of the group (e.g., 'Organization', 'Department', 'Team', 'Project'). |

## 8. Entitlement Group Assignments (`entitlement_group_assignments.parquet`)
Represents the membership of Identities in Entitlement Groups.

| Column Name | Data Type | Description |
|---|---|---|
| `identity_id` | String (UUID) | Foreign key to the Identity. |
| `ent_group_id` | String (UUID) | Foreign key to the Entitlement Group. |
| `assignment_status` | String | Status of the membership (e.g., 'Active'). |

## 9. Entitlement Group Relations (`entitlement_group_relations.parquet`)
Represents the hierarchical relationships between groups (Parent-Child).

| Column Name | Data Type | Description |
|---|---|---|
| `parent_ent_group_id` | String (UUID) | Foreign key to the Parent Group. |
| `child_ent_group_id` | String (UUID) | Foreign key to the Child Group. |

## 10. Group Entitlements (`group_entitlements.parquet`)
Represents the permissions assigned to a group (Role-Based Access).

| Column Name | Data Type | Description |
|---|---|---|
| `ent_group_id` | String (UUID) | Foreign key to the Entitlement Group. |
| `entitlement_id` | String (UUID) | Foreign key to the Entitlement granted to the group. |

## 11. Enumerated Types (Reference)

### Identity Status
*   `Active`
*   `Inactive`
*   `Leave`

### Identity Type
*   `Human`
*   `Service Account`

### Application Criticality
*   `Low`
*   `Medium`
*   `High`
*   `Critical`

### Resource Connection Type
*   `Direct`
*   `JDBC`
*   `API`
*   `FlatFile`

### Resource Type
*   `Platform` (e.g., AD, AWS)
*   `Application` (Standard business apps)

### Assignment Type
*   `Birthright` (Automatically assigned based on rules/roles)
*   `Adhoc_Anomaly` (Randomly assigned, potentially toxic/anomalous)

### Group Type
*   `Organization` (Root level)
*   `Department` (e.g., Engineering, HR)
*   `Team` (Sub-groups within departments)
*   `Project` (Cross-functional, ad-hoc)
